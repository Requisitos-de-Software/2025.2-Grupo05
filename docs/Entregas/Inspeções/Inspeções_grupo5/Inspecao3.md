## Lista de Inspeção - Especificação Suplementar 


A tabela 3 mostra a lista de verificação da técnica.

| O github pages possui: | Avaliação | Versão, Data e hora da avaliação |
| :---- | ----- | :---: |
| 1 – O objetivo desta Especificação Suplementar está especificado na seção Introdução?                            | | |
| 2 \- A seção Escopo contém uma breve descrição do escopo e a qual Projeto ele está associado?               | |  |
| 3 \- A subseção Definições, Acrônimos e Abreviações está alinhada ou faz referência ao Glossário do projeto?     | | |
| 4 \- A seção Referências lista todos os documentos mencionados na Especificação Suplementar, incluindo título, data e organização da publicação?  | |  |
| 5 \- A seção Utilidade inclui requisitos para o tempo de treinamento necessário para que os usuários se tornem produtivos?   | |  |
| 6 \- A seção Funcionalidade aborda requisitos funcionais que são expressos no estilo de idioma nativo e estão organizados por recurso, usuário ou subsistema?  | | |
| 7 \- A seção Confiabilidade especifica a porcentagem de Disponibilidade do sistema? | | |
| 8 \- O requisito de Confiabilidade inclui o MTBF (Mean Time Between Failures), especificado em horas, dias, meses ou anos? |  |  |
| 9 \- A seção Desempenho esboça as características do desempenho do sistema, incluindo tempos de resposta específicos e o rendimento do processamento (transações/segundo)? | | |
| 10 \-Os requisitos de Suportabilidade indicam padrões de codificação, convenções de nomenclatura ou bibliotecas de classe para aprimorar a capacidade de manutenção? | | |

## Lista de Inspeção - Caso de uso  

A tabela 4 mostra a lista de verificação do caso de uso.

| O github pages possui: | Avaliação | Versão, Data e hora da avaliação |
| :---- | ----- | :---: |
| 1- O sistema é representado por um retângulo? |  |  |
| 2- O sistema possui um nome? |  |  |
| 3- Tudo o que ocorre dentro da aplicação está dentro do retângulo? |  | |
| 4- Os atores estão posicionados fora do sistema? | | |
| 5- Os atores primários (iniciam a utilização do sistema) estão agrupados à esquerda do sistema? |  |  |
| 6- Os atores secundários (reagem as ações dos atores primários) estão agrupados à direita do sistema? |  |  |
| 7- Os casos de uso são representados por uma oval? | | |
| 8- Os casos de uso representam uma ação que realiza uma tarefa dentro do sistema?  |  |  |
| 9- As descrições em um caso de uso começam com um verbo? |  |  |
| 10- Cada ator interage com pelo menos um caso de uso do sistema? |  |  |
| 11- Os relacionamentos de associação (quando ocorre uma comunicação ou interação simples) é simbolizado como uma linha sólida? |  |  |
| 12- Os relacionamentos de inclusão (quando ocorre uma dependência entre um caso de uso baso e um incluído) é simbolizado como uma seta tracejada? | |  |
| 13- Os relacionamentos de extensão (quando ocorre necessidade de estender o caso de uso base) é simbolizado como uma seta tracejada? |  |  |
| 14- Os relacionamentos de generalização (quando um caso de uso compartilha das características de outro, acrescentando algo novo) é simbolizado como uma seta sólida com ponta branca? |  | |
| 15- Os pontos de extensão são versões detalhadas dos relacionamentos de extensão?  |  |  |
